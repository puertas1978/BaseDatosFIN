2. Relacional Data base. Conceptual design

mcmurryjulie. Pixabay (CC0)

Do you know what a database is?

If you think that a database is a collection of data, you're not far off.

In fact, we can say that:


Database = Data Warehouse

Now we're going to move on to understand a somewhat more formal definition of what a database is.

A database is a comprehensive, non-redundant set of structured data organized independently of its use and implementation on a machine, accessible in real-time, and compatible with concurrent users with different information needs.

Let's break down this definition so you can understand it better:

  • Comprehensive non-redundant set of data. We can say that it's a collection of non-repeated data.
  • Structured and organized independently of its use and implementation on a machine. In other words, the information is organized in a way that anyone can use it.
  • Accessible in real-time and compatible with concurrent users. Meaning, it's data that can be accessed in real-time and by many users simultaneously

Do you find it easier now? I bet you do!
